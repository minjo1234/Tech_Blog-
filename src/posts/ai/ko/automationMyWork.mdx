Claude Token 관리를 위해 Context를 읽어오는게 아닌 프로젝트 맞춤형으로
커스텀한 Claude Skills를 만들도록 하였다.

근데 사용하다보니 나한테는 원래 토큰 관리에 불과했더 이 녀석이 자동화를 위함이라는 것을

Claude Skill vs Claude Sub Agent

Jo의 ClovirONE 프로젝트 적용 예시
만들면 좋을 Skills:

ClovirONE-Development-Standard.md - 전체 개발 규약
VMware-Integration-Guide.md - vSphere API 사용 패턴
Frontend-Component-Style.md - Vue 컴포넌트 구조

만들면 좋을 Sub Agents:

Infrastructure Agent - VM 배포, 네트워크 설정
Monitoring Agent - 메트릭 수집, 대시보드 최적화
Security Agent - 인증/인가, 보안 정책
Database Agent - MariaDB 쿼리 최적화, 스키마 설계

Jo의 이해가 정확해! 이 구조로 가면 token도 절약하고, 일관성도 유지하고, 전문성도 확보할 수 있어.
혹시 지금 당장 만들어보고 싶은 Skill이나 Sub Agent 있어? 실제로 작성해볼 수 있을 것 같은데!

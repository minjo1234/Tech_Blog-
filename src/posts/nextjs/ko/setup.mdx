## 취지
---

1.기존의 기술 블로그를 tistory, velog, github.io 등등 플랫폼을 이용하였는데 간편히 사용하였지만, 직접 만들면서 커스터마이징한 기능을 넣고 싶다는 생각이 들었다.

2.Next.js 학습 (사내에서 Freemarker라는 템플릿 엔진을 사용하여 개발을 진행하다보니,
페이지마다 로드 속도의 차이가 많게는 10초이상 존재하였고(ex 지도페이지) 재로드가 많이 필요한 경우에 속도가 오래 걸렸기에,
이를 내가 먼저 학습하여 해결할 방법을 찾고자하였고, 그에 맞는 해결책이 Next.js라고 생각이 들어 미리 학습용 블로그를 만들어 보려고한다.)

## 기능정리
---

**필수 기능**

- 게시글 상세 페이지
- 게시글 목록 페이지
- 댓글 기능
- 포트폴리오 페이지(이력 관리)

**부가 기능**
- 목차 사이드바 (TOC, Table of Content)
- 다크/라이트 모드
- 최상단 최하단이동
- 게시글 주소 복사
- 검색 엔진 최적화(SEO)
- reading-time parsing
- scroll progress status bar

## 기술 스택 정하기
---

### 1.Framework: Next.js


Next.js는 페이지별로 CSR, SSR, SSG 등이 가능하며 SSR처럼 서버에서 렌더링하지만 **Streaming, Prefetching, Client-side Transition** 등의 부가 기능을 통해 > **앱처럼 부드럽고 빠른 사용자 경험(UX)**을 제공한다는 장점이 있다.

Version : Next.js v15.5.4

> 또한, 현재 **프론트엔드 시장에서 가장 주목받는 프레임워크** 중 하나이기 때문에 선택하게 되었다.

### 2.Design: Tailwind CSS

Bootstrap을 이용하고 있지만 빠르고 일반적인 디자인에는 편리하지만, 커스터마이징에는 범위가 한정적이였으며, 추가적인 css 파일이 빈번하게 생기곤 하였다.
Tailwind는 학습에 대한 초기학습량은 Bootstrap 보다는 많지만 이후에 매우 자유롭게 스타일이 가능하다는 이점때문에 선택하게 되었다..

Version : tailwindcss@4.1.14

### 3.글작성 : MDX

프로젝트 내부 디렉토리에 mdx 파일을 저장할 예정이고, 수정과 삭제에 대한 기능을 추가하지 않으려고 한다. (나만 수정하고 삭제하면 되기 때문이다.)
기본적인 `md`에 jsx 문법을 추가로 사용할 수 있는 `mdx` 파일을 선택하였습니다.
